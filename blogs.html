Frequently Asked Questions in Basic javascript Concept

1.var: The var keyword declares a function-scoped or global variable. Function-scoped means that the variable is only available within the function it was declared in. Global variables are available throughout your entire code.
2.let: The let keyword declares a block-scoped local variable.Block-scoped means that the variable is only available within the block it was declared in
3.constThe const keyword declares a block-scoped.a variable that can't be reassigned Constants are also called “immutable variables”, but that's a bit of a misnomer since they are actually variables - just ones that can't be reassigned.

Today, there are two common methods of declaring a function in JavaScript:
Arrow functions and Regular functions .
The first and most obvious difference between arrow functions and regular functions is their syntax. Not only do they look different, but arrow functions also provide an implicit return shorthand and allow parenthesis around a single argument to be omitted. Regular functions can be used as constructors, using the new keyword. Yet another consequence of the lexical resolution of this inside arrow functions is that they cannot be used as constructors. Using new with an arrow function results in a TypeError.
Another difference is the binding of the arguments object. Unlike regular functions, arrow functions don't have their own arguments object. A modern alternative that circumvents this limitation is the usage of rest parameters.

  • The forEach() method is the most similar to the for loop. The function inside the parentheses of the .forEach() is simply executed "for each" element of the array.
  • The map() is a lot like .forEach(), but it very helpfully creates and returns a new array. Use .map() whenever you need to update data inside an array (by mapping over it!).
  • The filter() loops through (or iterates) through data, and filters out data that doesn't match the criteria that we set. We define what those criteria are through a truth test inside a function.
  • The find() method returns the value of the first element that passes a test. The find() method executes a function for each array element. The find() method returns undefined if no elements are found.
